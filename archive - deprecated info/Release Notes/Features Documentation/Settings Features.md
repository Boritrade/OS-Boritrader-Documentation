# Boritrader Features List & Documentation: Settings

**Boritrade, LLC**  
**Last Updated:** April 2, 2025

This document provides a comprehensive guide to the features available on the Settings page of Boritrader. It explains how users can manage personal details, update security credentials and API keys, handle subscription settings, and more. Additionally, it notes sections that are currently deprecated, ensuring clarity on which features are active.

---

## 1. Profile Information

**Description:**  
The Profile Information section allows users to update their personal details. Although fields such as full name, address, and date of birth are provided, they remain optional and are not used in day-to-day operations. This section is designed to let users personalize their account without impacting the core functionality of the platform.

**Summary:**
- **Purpose:** Update personal details (full name, address, date of birth).
- **Usage:** Fields are optional and not operationally used.
- **User Action:** Fill in or update profile information as desired.

---

## 2. Account Settings

**Description:**  
The Account Settings section focuses on maintaining account security. Users are prompted to change their password by first entering their current password and then choosing a new one. On-screen guidelines ensure that the new password meets all necessary security standards, reinforcing the overall protection of user accounts.

**Summary:**
- **Purpose:** Update account security details.
- **Usage:** Change password by providing the current password and a new password.
- **User Action:** Follow on-screen guidelines to ensure the new password is secure.

---

## 3. Trading Preferences (Deprecated)

**Description:**  
Originally, the Trading Preferences section was intended to let users toggle between different trading modes (e.g., switching between Auto Purchase and Notification Only, or between Live Trading and Paper Trading). Although these settings still appear on the interface, they are not yet active and do not affect trading behavior at this time.

**Summary:**
- **Original Function:** Toggle between different trading modes.
- **Current Status:** Displayed in the interface but not functional.
- **User Action:** No action required as these settings are deprecated.

---

## 4. Notification Preferences (Deprecated)

**Description:**  
The Notification Preferences section was designed to allow users to choose how they receive alerts and updates. Similar to the Trading Preferences, this section remains visible on the Settings page but is currently inactive. 

**Summary:**
- **Original Function:** Customize alerts and update methods.
- **Current Status:** Present but not in use.
- **User Action:** No modifications needed.

---

## 5. API Keys Section

**Description:**  
This section provides a secure area for users to manage their Binance API credentials. Users can update their API key, secret, and region settings, as well as indicate if they are using a testnet key. Clear helper text is provided to guide users through the process of creating and managing these keys, ensuring that they have the correct permissions set up for secure trading.

**Summary:**
- **Purpose:** Enter and update Binance API credentials.
- **Usage:** Update API key, secret, region settings, and testnet key option.
- **User Action:** Follow helper text for accurate API key creation and management.

---

## 6. Subscription Management

**Description:**  
The Subscription Management section gives users the tools to control their subscription plan. Whether updating the plan, canceling the subscription, or modifying billing details, users can access these functions directly through the interface. The section also provides a “Sync Subscription” option to refresh the subscription status, ensuring that users always have current access details.

**Summary:**
- **Purpose:** Manage your subscription plan.
- **Usage:** Update plan, cancel subscription, or change billing details.
- **User Action:** Use the “Manage Subscription” and “Sync Subscription” buttons for updates.

---

## 7. Account Deletion

**Description:**  
The Account Deletion feature is designed for users who wish to permanently remove their account and all associated data. This section includes a clear warning message to ensure that users understand the irreversible nature of this action. Users must confirm their decision when prompted, ensuring deliberate action before proceeding with deletion.

**Summary:**
- **Purpose:** Permanently delete the account and associated data.
- **Usage:** A clear warning is provided due to the irreversible nature.
- **User Action:** Review the warning and confirm deletion when prompted.

---

## 8. Security Measures

**Description:**  
Security is a critical focus on Boritrader. This section details the robust, industry-standard security measures that protect user data. It highlights the use of advanced encryption and hashing techniques for safeguarding sensitive information, such as passwords and API keys. These measures ensure that user credentials are securely stored and that the platform remains resilient against potential threats.

**Summary:**
- **Purpose:** Outline the platform's security measures.
- **Usage:** Advanced encryption and hashing for safeguarding sensitive data.
- **User Action:** No direct action; serves as an assurance of the platform's security.
