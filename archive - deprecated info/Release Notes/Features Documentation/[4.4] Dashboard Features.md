# Boritrader Features List & Documentation: Dashboard

**Boritrade, LLC**  
**Last Updated:** April 2, 2025

This document details every feature of the Boritrader dashboard, including user instructions. It serves as a comprehensive guide for understanding and communicating the value of the app’s functionality, covering key elements from the header to interactive dashboard components and detailed button functions.

---

## 1. Header Section Breakdown

The header is a central element providing quick access to account information and primary navigation. It is divided into several key areas to ensure both functionality and intuitive design.

### 1.1 User & Subscription Buttons

**Description:**  
The User Button displays the current user’s username and unique ID and acts as a quick route back to the dashboard. In addition, the Subscription Status Button shows the current subscription status, complete with a colored indicator that reflects the state of the subscription. This dynamic element offers real-time visual feedback (with colors such as green for active, red for canceled, yellow for past-due, and gray for inactive) and links directly to subscription management in the settings.

**Summary:**
- **User Button:**
  - Displays username and unique ID.
  - Clickable element that routes back to the dashboard.
  - Accompanied by a user icon for easy recognition.
- **Subscription Status Button:**
  - Shows current subscription status (or “N/A” if not available).
  - Includes a colored circle indicator for visual feedback.
  - Clicking navigates to the settings page for subscription management.

---

### 1.2 Header Logo

**Description:**  
Centrally placed in the header, the logo reinforces the brand identity and acts as a navigational shortcut. Clicking the logo returns the user to the main dashboard, serving both functional and branding purposes.

**Summary:**
- Centrally placed for brand reinforcement.
- Clickable, routing back to the dashboard.
- Serves as a visual anchor for the interface.

---

### 1.3 Navigation Menu

**Description:**  
The navigation menu provides easy access to essential sections of the dashboard, such as Home, Settings, and Logout. Each link is enhanced with an icon and clear labeling, ensuring that users can navigate the platform with ease.

**Summary:**
- Provides links to Home, Settings, and Logout.
- Each link features a clear icon and label.
- Simplifies navigation across key dashboard areas.

---

### 1.4 Additional Technical Elements

**Description:**  
Beyond user-facing components, the header includes technical elements that ensure a robust experience. Integration with Sentry facilitates error tracking, while a favicon and base CSS styles contribute to a consistent design across browser tabs and sessions.

**Summary:**
- **Sentry Integration:** Error tracking for performance reliability.
- **Favicon:** Maintains brand consistency in browser tabs.
- **Base Styles:** Consistent design through linked CSS.

## 2. Dashboard Core Features

This section covers the main interactive and informative components of the dashboard, each designed to enhance the user’s trading experience.

### 2.1 Trial Notification Banner

**Description:**  
The Trial Notification Banner appears for users in trial mode, displaying the trial’s expiry date along with a prompt to subscribe. It provides clear, actionable information and disappears automatically once the user activates their subscription.

**Summary:**
- Displays when the user is in a trial mode without an active subscription.
- Clearly states the trial’s expiry date.
- Includes a “Subscribe now!” link for upgrading.
- Disappears automatically once a subscription is activated.

---

### 2.2 Trade Log Toggle & Trade Logger Card

**Description:**  
Users can manage the visibility of their trade logs using a toggle button that switches between “Hide Log” and “Show Log.” The Trade Logger Card, marked as “BETA,” displays real-time updates of trading activity, allowing users to monitor their trade history effectively.

**Summary:**
- **Toggle Button:** Switches between “Show Log” and “Hide Log” to control visibility.
- **Trade Logger Card:** Displays real-time trade activity logs (BETA version).
- Enhances transparency and performance tracking.

---

### 2.3 Status Circle & Real-Time Clock

**Description:**  
The status circle with glow animations, paired with a digital clock, offers a quick visual summary of system activity. This feature provides users with immediate insights into both market status and algorithm performance.

**Summary:**
- Displays dynamic status updates with glow animations.
- Includes a real-time digital clock.
- Serves as a quick reference for system and market activity.

---

### 2.4 Portfolio Summary Card

**Description:**  
The Portfolio Summary Card provides a snapshot of the user's crypto holdings. It lists all coins with icons and amounts, highlights the top three assets based on value, and shows both current and available balances, giving users immediate insights into their investments.

**Summary:**
- Lists crypto holdings with corresponding icons and amounts.
- Highlights the top three assets based on value.
- Displays current and available balance information.

---

### 2.5 Trading Algorithms Section

**Description:**  
This section lists available trading algorithms and provides interactive controls. Users can enable/disable an algorithm with a toggle and use buttons to configure, view status, or access detailed information, allowing for a high level of control over trading strategies.

**Summary:**
- Lists available trading algorithms.
- Toggle switch to enable or disable each algorithm.
- Buttons to **Configure**, **View Status**, and access **Details**.

---

### 2.6 Advanced Algorithms Section (Dropdown)

**Description:**  
Targeted toward advanced users, this dropdown section reveals additional algorithm details, including risk levels. It allows users to manage more sophisticated trading strategies with ease and clarity.

**Summary:**
- Similar to the basic algorithms section but for advanced settings.
- Dropdown toggle to reveal or hide extra details (e.g., risk levels).
- Designed for users requiring deeper algorithmic control.

---

### 2.7 Popup Windows for Configuration & Details

**Description:**  
Popup windows offer dynamic forms for configuring algorithm preferences and viewing detailed insights. These windows ensure that users can quickly access and update settings without leaving the main dashboard.

**Summary:**
- **Configure Popup:** Dynamic form to update algorithm settings.
- **Details Popup:** Displays in-depth insights and performance data.
- **Status Popup:** Provides real-time notifications and historical data review.

---

### 2.8 Real-Time WebSocket Integration

**Description:**  
Real-time WebSocket integration pushes live notifications directly to the dashboard. This feature ensures that algorithm events and trade updates are received instantly, keeping users well-informed to react quickly in the market.

**Summary:**
- Establishes a live WebSocket connection.
- Pushes real-time notifications and trade updates.
- Enables prompt responses to market changes.

---

### 2.9 Interactive and Responsive UI

**Description:**  
The dashboard’s interactive UI features animated indicators, a real-time clock, and responsive buttons. This design ensures a smooth, intuitive experience, allowing users to engage actively with every element on the platform.

**Summary:**
- Features dynamic elements such as animated indicators and a live clock.
- Responsive buttons and toggles for smooth interactions.
- Intuitive layout for enhanced user engagement.

---

### 2.10 Algorithm Notification & Management System

**Description:**  
This system fetches and displays the latest algorithm signals and notifications. It also includes a function to clear historical notifications, keeping the display organized and allowing users to monitor trading signals efficiently.

**Summary:**
- Displays the latest algorithm signals and notifications.
- Provides a clear function to remove past notifications.
- Helps keep the signal display tidy and focused.

---

## 3. Detailed Button Functionality

This section outlines the specific functionalities of key buttons on the dashboard, ensuring that users understand how to interact with detailed components.

### 3.1 View Status Button

**Description:**  
The View Status Button opens a detailed signal view that displays current trading signals—including ticker symbols, statuses (BUY, SELL, or DO NOTHING), and timestamps. It also includes a collapsible section for historical data and a control to clear notifications, helping users maintain an organized overview of algorithm performance.

**Summary:**
- Opens a view displaying real-time trading signals.
- Shows ticker symbol, current status, and timestamp.
- Includes a collapsible section for historical signals.
- Provides a control to clear all notifications.

---

### 3.2 Configure Button

**Description:**  
The Configure Button launches a popup form that allows users to customize trading algorithm settings. Users can update parameters such as tickers, trade quantity, and execution frequency, and then save their preferences to personalize their trading strategy.

**Summary:**
- Opens a popup form for algorithm configuration.
- Allows updating of tickers, quantity, and execution frequency.
- Enables users to save personalized trading settings.

---

### 3.3 Details Button

**Description:**  
The Details Button provides an in-depth overview of a selected trading algorithm. This detailed view includes a comprehensive description of the algorithm’s purpose, core functionalities, and a disclaimer, offering transparency and educational insight into its operation.

**Summary:**
- Provides an in-depth description of the selected algorithm.
- Details the algorithm’s purpose, functionalities, and disclaimer.
- Aids in understanding the algorithm’s logic and best-use cases.
